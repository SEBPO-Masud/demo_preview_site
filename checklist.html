We maintain a Checklist after completing development process

# Checklist
1 CSS has passed validation, Tools: https://jigsaw.w3.org/css-validator/
2 HTML has passed validation, Tools: https://validator.w3.org/
3 Is Website Performance Testing and Monitoring by GTmetrix: https://gtmetrix.com/
4 Make sure that the landing page developed correctly as per mock.
5 Export the images 2x of it's sizes
6 The max image size to be used is 500k, below 100kb is standard.
7 All the alt text will be used where applicable.
8 Font to be used PetcoCircular Bold and Regular.
9 Check that images are being resized correctly and fit for purpose (i.e. optimised for size and speed)
10 Alignment and spacing are consistent throughout the site
11 Remove all placeholders, Lorem Ipsum, dummy copies, and unnecessary data from the website content.
12 Check the availability of 404 pages.
13 Verify links destination, Remove hidden links and External links open in a new tab
14 If there is a Form submission and confirmation. Tools: Use the “Fake Filler” Chrome extension to do this faster Data collection
15 Buttons: double-check colors for CTAs, normal, visited, hovered, active, disabled, etc.
16 HTML and CSS class declaration rules : .citrus-brandName-element ex: .ctrus-purebites-carousels
17 Need to check tagging name and links from the tagging doc.
18 Need to check tagging links are working perfectly.
19 Ensure that the correct tagging name added along with the links.
20 Wrap up the full block/card with the anchor link.
21 Ensure link validity and Remove broken links: https://www.deadlinkchecker.com/
22 Page title is added and relevant
23 Add alt text attributes for images
24 Maintain the hierarchy of the headings of h1, h2, h3… and p tags.
25 Each page has been run through Google Page Speed and ensure the score for both desktop and mobile is ‘good’: https://pagespeed.web.dev/
26 Cross-browser testing: Firefox, Chrome, Safari, Edge, Opera (at least the last three versions), Tools: https://www.browserstack.com/
27 Cross-system testing should cover Windows, Mac, Android, and iOS (at least the last three versions and 3 device) Tools: https://www.browserstack.com/
28 Make sure not to keep the unnecessary white spaces.
29 Need to check raw-js, pov-rotator, splide-js for tab and carousel section.
30 For multiple carousels, all the carousel height should be same as per mock.
31 Check the carousels, arrows and dots are working properly.
32 Make sure to add the elipsis css styles with max-height: 4em; for product scroller p tags to keep … end of the line
33 Make sure to use the (width: 2.2em;) style for splide arrows
34 Ensure that all notes from instant messaging conversations, any confirmations, conversation summaries, and information related to guideline breaches are added to the task ticket when delivering the project.
35 Have all the required files, including previews, production files, text/sheets, fonts, required assets, and source files, been uploaded to the online and organized appropriately?
36 Ensure that all deliverable required URLs are added to the ticket correctly and point directly to the respective file locations.
37 Verify that the ticket is assigned to the corresponding task reporter.

1 Compress all the images to reduce the file size from https://tinypng.com/
2 Remove the html tag, header tag and the links within it, body tag.
3 Add all the files (html, css, js) within one file.
4 Follow the file sequences as <Style> all the style goes here </Style>, <markup>all the html goes here </markup> and <script>all the javscript n library goes here</script>
